# Allocation Project Documentation

## Project Overview
Architecture notes for this codebase: **why** non-obvious choices exist, not tutorials for things the code already shows.

## Publications and derived data

- **`memberThemesRefreshCoordinator`**: One debounced `MemberThemes` refresh per `themeId` is shared by `theme`, `organizations`, `members`, and **`member`** publications so concurrent subscribers do not multiply watchers or redundant full-table fetches. Kiosk `RemoteVoting` uses `useMember` + the `member` subscription instead of the full `members` list for the same reason.
- **Static (one-shot) orgs and members**: Shared publication-shaped data is built under **`imports/server/themeDataLoaders/`** — **`memberThemeLookup.ts`** (`buildMemberThemeLookupMap`) and **`organizationPublicationFrame.ts`** (`getOrgPublicationFrame`, `refreshOrgObserverParamsInPlace`); **`imports/server/publications/organizations.ts`** and **`members.ts`** import those modules (not the snapshot folder). **`imports/server/snapshots/`** holds only **`fetchOrgsSnapshot`**, **`fetchMembersSnapshot`**, and **`fetchMemberSnapshot`**, which call the same frame/lookup helpers the publications use. **`ValidatedMethod` definitions that call those fetch helpers** live in **`server/api/methods/staticSnapshotMethods.ts`**: **`new ValidatedMethod({ name: … })` registers the RPC when that module is evaluated** (side effect at import time); the **`export const …Method`** names are only for tests and introspection, not what wires DDP. The module is imported from **`server/main.ts`** for normal **`meteor run`** / production, and **`server/api/methods/*` tests** **`import "./staticSnapshotMethods"`** so those RPCs exist in the server test harness—**`imports/api/methods/*` never imports `imports/server/`** (Meteor forbids client bundles from pulling `imports/server`). Client hooks call the same RPC names via **`imports/api/methods/staticReadCalls.ts`** (`Meteor.callAsync` + narrowers; **`OrgsListSnapshot`** and **`MembersListSnapshot`** are exported there as the RPC result shapes so isomorphic code never imports server-only modules for typing—**`imports/server/snapshots/organizations.ts`** may **`import { type OrgsListSnapshot }`** from that file for return typing only). Read method names: **`organizations.list`**, **`members.list`**, **`members.get`**. Client code uses **`useStaticOrgs`** and **`useStaticMembers`** / **`useStaticMember`** (under **`imports/api/hooks/static/`**) so kiosk voting, pledges, and presentation **Orgs / TopOrgs / Results** slides can load without subscribing to **`organizations`**. **Kiosk and presentation shells** still use reactive **`useTheme`** and **`useSettings`** where flags and navigation matter; **allocation** and **PledgesOverlay** keep **`useOrgs`** + **`usePledgeAnimationQueue`** only while those components are mounted. There is no static read path for theme, presentation settings, or messages in this phase. **`fetchOrgsSnapshot`** builds **`topOrgs`** with **`filterTopOrgs`** on **`OrgTransformer`** output (same ordering as the theme publication stack), not from a persisted **`topOrgs`** field on the raw theme document. **`OrgTransformer`** returns **`OrgDataWithComputed`** with explicit numeric defaults for optional **`Organization`** money fields so snapshot and publication paths stay typed without **`as`**; **`OrgWithComputed`** is **`OrgDataWithComputed & Record<string, unknown>`** so **`registerObserver`**'s **`Record<string, unknown>`** contract stays satisfied.
- **`members` publication**: Membership is driven by a join through `MemberThemes`. When that join changes, the publication diffs IDs before touching the `Members` observer so dropped members emit **`removed`** immediately; the same transformer is reused so the client does not keep stale rows.
- **`organizations` publication**: A single org change can reorder or rescale the **global** chronological pledge walk for every org. Org `observe()` callbacks are wrapped in a **debounced full re-publish** so `matchedAmounts` and downstream org summaries stay coherent without re-implementing the walk per callback.
- **Refresh failures**: `catch` blocks in `themes.ts`, `organizations.ts`, and `members.ts` log through `publicationLog` (`LogCategories.Publication`, `mirrorToConsole: true`, `LogModels.*`) so broken refreshes show in server logs and in `Logs` when persist succeeds. Extra context that is not part of `LogContext` (e.g. a member id) goes under **`meta`**, not top-level fields.

## Pledge matching and leverage

- All matched-pledge math lives in **`imports/lib/allocation/pledgeMatching.ts`**. Do not re-implement `pledge.amount * matchRatio` elsewhere: it ignores the shared leverage pool, mixed `pledgeType` ratios, and partial-match truncation.
- **One pool per theme**: Eligible pledges across orgs share a single chronological queue (`createdAt`, then `_id`). Pool exhaustion can zero or partially match a later pledge regardless of org.
- **`calculatePledgeMatches` persists nothing**; it returns `matchedAmounts` + `remainingLeverage`. Publications recompute and pass **`matchedAmounts`** into transformers on each tick so edits/removals re-walk the queue without storing derived match state in Mongo.
- **Ratios**: `MatchPledge.pledgeType` selects `Theme.matchRatio` vs `Theme.inPersonMatchRatio`. Both are **total multipliers** (e.g. 2 ⇒ pledge + leverage bonus totals 2x pledge dollars from the pool perspective for matching logic as implemented). **`matchMultiplierForPledge`** / **`formatMatchRatioFromMultiplier`** in **`imports/lib/allocation/pledgeMatching.ts`** resolve the multiplier and the human **n:1** label (**m** → **(m-1):1**, e.g. 2 → `1:1`); admin **Pledges** table uses these exports. **`PledgeComplete`** uses the same **(m-1):1** wording inline.
- **Runner-up rule** is centralized in **`isOrgEligibleForLeverage`** for both queue construction and `pledgeTotalForOrg` — keep it in sync; do not duplicate.
- **`pledgedTotal` vs `pledgeMatchTotal`** in `ThemeTransformer`: raw pledged dollars vs total **leverage bonus consumed**; they differ whenever ratios ≠ 2, in-person pledges exist, or the pool truncated matches. **Why**: different surfaces need a footer that matches per-org rows vs a leverage equation that still closes.

## Feature flags vs runtime (presentation)

- **Flags** (Theme / PresentationSettings forms): “Is this capability allowed for this theme or presentation?” e.g. `Theme.inPersonPledgeActive`, `PresentationSettings.useKioskChitVoting`.
- **Runtime** (`PresentationPane`): “Is the live show doing this now?” e.g. `topupsActive`, `chitVotingActive`, `fundsVotingActive`, `currentPage`.
- **In-person pledges**: `/pledges/inperson/:themeId` is intended to work only when **both** `theme.inPersonPledgeActive` and `settings.topupsActive` are true; each false case gets its own messaging. **Server** `OrganizationMethods.pledge` enforces only the **theme flag** (same idea as standard pledges and runtime gating on the client) — keep that split deliberate.
- **In-person URL uses Mongo `_id`**: obscurity for a short window, not a secret; QR is **admin/settings only**, not on the presentation surface.
- **Pledge thank-you UI**: **`PledgeComplete`** (**`imports/ui/components/PledgeComplete/index.tsx`**) is the single thank-you screen for kiosk remote pledges (**`matchKind="standard"`** → **`Theme.matchRatio`**) and in-person (**`matchKind="inPerson"`** → **`Theme.inPersonMatchRatio`**). Layout and styles match the original kiosk confirmation (**`justify-content: space-around`**, centered column, plain **`<p>`** body with **1.25rem** white copy, **`AmendVoteButton`** styling); ratio line + optional leverage-pool note are shared. No sign-out on this screen. **`kioskSurfaceThemeOptions`** (module-level **`ThemeOptions`** in **`imports/ui/layouts/KioskLayout/index.tsx`**) is merged in **`KioskLayout`**'s **`ThemeProvider`**, not inside **`PledgeComplete`**. Remote pledges sit under **`Kiosk`**'s **`PageFader`** (**`max-width: 100vw`**, **`min-height: 100%`**, **`width: 100%`**); in-person thank-you wraps **`PledgeComplete`** in **`InPersonPledgeForm`** with the same box so **`height: 100%`** / **`space-around`** get the same containing block as the kiosk route.

## Terminology (legacy cleanup)

- **pledge**, **crowdFavorite** (amount in `Organization.topOff`; `OrganizationMethods.crowdFavorite` sets gap = ask - votes - save - starting funds - **`pledgeTotalForOrg`** after **`calculatePledgeMatches`**, then **`topOff` = min(gap, remainingLeverage)**), **leverage** — see schema `// NOTE` on legacy names (`topOff`, `topupsActive`, etc.). **`CrowdFavoriteButton`** calls the method; do not hand-update `topOff`.

## Chit voting: two storage shapes

- **Manual** (`useKioskChitVoting` false): aggregate **`OrgChitVoteSummary`** on `Organization.chitVotes`; count beats weight; weight uses `Theme.chitWeight`.
- **Kiosk** (true): per-member **`ChitVote`** rows on `MemberTheme.chitVotes`; **`OrgTransformer`** aggregates. Do not conflate the two models.

## Logging (`imports/lib/logging/`)

- Methods under `imports/api/methods/` ship to the client; logger utilities must stay **outside `imports/server/`** so the bundle can load them.
- **Scope**: presentation / theme-scoped events only — **`themeId` is required** on `LogContext`. Cross-theme or non-presentation work uses **`consoleLog`** from **`/imports/lib/logging`** (not forced `themeId`).
- **`env`** / **`consoleLog`** live in **`imports/lib/logging/index.ts`**. **`env`** is **`"production"` | `"test"` | `"development"`** from Meteor; **`consoleLog`** no-ops when **`env === "production"`**. App code imports both from **`/imports/lib/logging`** instead of **`console.*`**.
- **Named loggers** in `loggers.ts` carry **`model`** as part of their contract; there is no generic “methods” logger. **`mirrorToConsole`** is per-call. **`Logs`** is server-insert only; clients use **`logs.purge`** / **`logs.recent`** with server-side validation.

## UI / tooling gotchas

- **Admin overview CSV exports** (**`ExportMemberVotes`**, **`ExportPledges`**, **`ExportChitVotes`**): render **`ExportCsvButton`** immediately; use **`disabled`** while **`useOrgs` / `useMembers`** subscriptions are loading. Do not gate the button on **`isEmpty(members)`** — a theme with zero members finishes loading with an empty array and would otherwise show **`Loading`** forever.
- **`/imports/ui/components` barrel**: re-exports shared UI, forms, dialogs, cards, etc. It does **not** re-export **`Buttons/*`**, **`Toggles/*`**, or **`Inputs/*`** — import those from **`/imports/ui/components/Buttons/...`**, **`/imports/ui/components/Toggles`**, and **`/imports/ui/components/Inputs`**. The barrel exposes **`ConfirmationModal`** (`ConfirmDelete`) and **`CustomConfirmationModal`** (`CustomConfirm`, rich `content`) as distinct names. At **`imports/ui/components/`** root only **`index.ts`** should live; each component is a subdirectory (e.g. **`PledgeComplete/index.tsx`**) exported through the barrel. **`NumberInput`** (**`imports/ui/components/Form/NumberInput.tsx`**) is exported next to **`Form`** / **`TextInput`**: MUI **`TextField`** with **`type="text"`** and **`inputMode`** (no native number spinners), filtered digits / optional decimal, **`emptyValue`** on blur when blank (default **`0`**), optional **`integer`** for whole numbers only; **`imports/lib/forms/numericFieldUtils.ts`** holds the filter helpers for **`NumberInput`** — keep one **`Form`** numeric control, not a second parallel **`TextField`** wrapper. **`ThemeSettingsForm`** wires **`NumberInput`** for all dollar fields (**`leverageTotal`**, **`consolationAmount`**, **`minLeverageAmount`**, **`minStartingFunds`**) and the integer ratio/chit fields; anything still on **`TextInput`** is plain text (e.g. **`question`**).
- **`TextInput` / `NumberInput` (typed RHF fields)**: Both are generic **`function`** components over **`TFieldValues extends FieldValues`** with **`name: FieldPath<TFieldValues>`** and **`useFormContext<TFieldValues>()`** so **`Controller`** does not need a **`name` cast**. **`ref`** is an optional prop (React 19 **`ref` as a prop** on plain functions) and is forwarded to MUI **`TextField`** **`inputRef`** — no **`forwardRef`** / **`as …Component`** workaround. **`onUpdate`** passes **`form` as `unknown`**; callers narrow (see **`MembersEdit`** **`handleInitials`**). **`NumberInput`** merges **`slotProps.htmlInput`** with **`mergeSlotProps`** from **`@mui/material/utils`** so an existing **`onBeforeInput`** runs before the numeric filter without manual **`SlotProps`** typing hacks.
- **TanStack Router pinned to `1.156.0`**: newer releases rely on `package.json` `exports` paths Meteor's bundler does not resolve ([meteor#13520](https://github.com/meteor/meteor/pull/13520)); bumping the router without fixing that breaks the build.
- **Nested `$pull` on pledge arrays**: `_id` `autoValue` on embedded pledge schema must **not** `unset()` for non-`$push` operators or **`$pull` by `_id` becomes `$pull: { pledges: {} }`** and wipes the array.
- **MUI v9**: Checkbox/Switch use **`slotProps.input`**; Stack/Grid flex props belong in **`sx`**, not loose DOM props (MUI forwards only a small allowlist to the root). For **`styled` from `@mui/material/styles`**, use **`styled("div")` / `styled("p")`** — there is no **`styled.div`** / **`styled.p`** proxy on the returned **`CreateMUIStyled`** factory (calling **`styled.p`** breaks the bundle / white screen).
- **React 19**: do not spread props that include **`key`**; pull `key` out and pass explicitly where MUI render props inject it.
- **`PresentationLayout`** (`/presentation/$id`, **`/simulation/$id`**): **`#presentationContainer`** is **`height` / `max-height: 100dvh`** with **`overflow: hidden`** so tall slides are clipped inside the shell instead of growing the document (no separate **`html`/`body`** scroll lock). Presentation and simulation routes wrap **`Loading`** in the same layout. **Results** caps the slide with **`height` / `max-height: 100dvh`**, **`gridTemplateRows: auto minmax(0, 1fr)`**, and **`overflow: hidden`** so many org rows do not grow the page. **`AwardCard`** fits **`org.title`** by binary-searching **`rem`** font size inside a flex **`TitleBox`** observed with **`ResizeObserver`** (min about **`0.55rem`**, max **`1.7rem` / `1.25rem`**).

- **`/pledges-overlay/$id`**: fullscreen stacks use **`position: fixed; inset: 0`** instead of **`100vw` / `100vh`** so width does not include the vertical scrollbar gutter (which forces horizontal scroll). Pledge copy is large fixed **`rem`** sizes again; overlay/text layers use **`overflow: visible`** so recognition text can paint past the viewport without ancestor clipping (still no `100vw`). The fireworks **`canvas`** wrapper keeps **`overflow: hidden`** so the Matter canvas does not spill layout-wise.
- **Presentation `PledgesOverlay`** (**`imports/ui/pages/Presentation/Allocation/PledgesOverlay/index.tsx`**): the queue-draining **`useEffect`** must **not** depend on **`pledges`** / **`queueItems` by reference** — **`useTracker`** returns new array identities often, so the effect would re-fire while **`isAnimating`** is still false and schedule duplicate **`setTimeout`** starts for the same queue head. Drive the effect off **`queueItems[0]?._id`**, a boolean **`pledgeReadyForHead`**, and **`isAnimating`**; mirror **`queueItems`/`pledges` into refs via `useLayoutEffect`** (not during render — **`react-hooks/refs`** disallows **`ref.current`** writes on the render path); the passive effect reads those refs; clear the long **`markProcessed`** timeout only when the queue head id changes (not on every unrelated tracker tick).

## Meteor / tests / scripts

- **ValidatedMethod**: calling **`.callAsync()` from inside another server method** can hang — prefer direct collection access in that case.
- **Root `scripts/`**: files with top-level side effects can run on Meteor startup; keep heavy scripts under `imports/` or gate on env.
- **Tests**: **`resetDatabase`** uses per-collection **`removeAsync({})`** instead of **`dropDatabase()`** to avoid Accounts races and slow index rebuilds. Shared server-test helpers live under **`imports/test-support/`**: **`fixtures/`** (theme + org factories), **`mocks/`** (frozen real-theme snapshots for leverage spread tests — **`Education`**, **`Democracy`**, **`Youth`**; not used in production), **`resetDatabase`**. Import fixtures from **`/imports/test-support`**; import mocks from **`/imports/test-support/mocks`** only (mocks pull **`imports/server/transformers`**). **`npm test`** runs **server** Mocha by default (no headless browser) unless you set **`TEST_BROWSER_DRIVER`** for client **`*.test.*`** files (see [meteor-mocha README](https://github.com/meteortesting/meteor-mocha/blob/master/README.md#run-app-tests)). Suites that need **full Mongo / methods / Collection2** (e.g. **`resetDatabase`**, **`ValidatedMethod.callAsync`**, schema **`autoValue`**) live under **`server/api/**`** (mirrors **`imports/api/methods`** and **`imports/api/db`**) so they never load in the client harness. Isomorphic **`imports/**/*.test.ts`** remain for logic that is safe on both sides (e.g. **`imports/lib/**`**). Tests that import **`imports/server/**`** (e.g. mocks that pull **`OrgTransformer`**) must live under **`server/`** or **`imports/server/`**, not **`imports/lib/`**, or the **client** test bundle will fail to load.
- **TypeScript**: **`npm run type-check`** runs **`tsc --noEmit`** (see **`tsconfig.json`**; **`ignoreDeprecations`: `"6.0"`** silences **`baseUrl`** deprecation until TS 7). Meteor packages without typings (e.g. **`meteor/aldeed:collection2`**, **`meteor/accounts-google`**) are declared in **`meteor-side-effect-modules.d.ts`** at the repo root so **`tsc`** resolves side-effect imports in **`server/main.ts`** and **`imports/api/db/index.ts`**.
- **Lint + types + tests**: **`npm run verify`** chains **`npm run lint`**, **`npm run type-check`**, and **`npm test`**. **`eslint.config.cjs`** starts with a standalone **`{ ignores: [...] }`** block (flat config global ignore) for **`.meteor/**`**, **`.coverage/**`**, **`node_modules`**, etc., so ESLint does not walk the Meteor bundler cache or Istanbul output. JSON uses **`eslint-plugin-jsonc`** for **`jsonc/no-dupe-keys`** only (no **`jsonc/indent`** — mixed 2-space and tab JSON would otherwise flood the run).
- **Publication load harness**: env-tunable scale in **`imports/server/test-support/publicationPipelineSimulation.ts`**; tests live in **`server/publicationPipelineSimulation.test.ts`** (server-only directory so the harness is not pulled into the **client** test bundle).

## Types and collections

- **`meteor/aldeed:collection2`** / **`meteor/aldeed:collection2/static`**: Meteor package ids, not npm; TypeScript does not ship typings for them. **`imports/types/collection2.d.ts`** declares those modules (side-effect imports) and augments **`Mongo.Collection`** with **`attachSchema`**. Same pattern as other **`imports/types/*.d.ts`** shims for Meteor or non-typed packages.
- **Generated types**: `npm run generate-types` → `imports/types/schema.ts`; collection vs embedded schemas differ (e.g. **`_id`** on collection-attached schemas only). **Embedded** `OrgChitVoteSummary` ≠ **`ChitVote`** on member themes — different purposes; name collisions across files are a footgun.
- **`Members` minimongo type is `MemberWithTheme`** (`MemberData` + optional **`theme`** from `members` / `member` publications) so client reads match published shape. **`MemberData`** remains the generated **Member** schema type from SimpleSchema. Import **`MemberWithTheme` from `/imports/api/db`** in UI and hooks — not from `imports/server/transformers/memberTransformer` (that path is server-architecture-only in Meteor and breaks the client test bundle).

## Kiosk / member login

- **`KioskVotingProvider` / `useKioskVoting`**: one session object for **both** funds allocations and chit tallies + saves + `member` + `unsetUser` (not funds-only). **`MemberLoginRequired`** vs **`RemoteVoting`**: URL without `member` uses the shared login form for chit/funds; pledges with a `member` query use **`RemoteVoting`** only.
- **`useFindMemberByCode`**: uses **`useStaticMembers`** so kiosk / in-person flows do not open the full `members` publication for lookup. **`MemberCodeLoginForm`**: folder + barrel under **`/imports/ui/components`**; shared kiosk + in-person login (**`notFoundStyle`** differs).
- **`imports/lib/`** layout: only **`index.ts`** at the root (selective re-exports); domain folders **`allocation/`** (pledge matching, leverage distribution, top-org ranking), **`logging/`**, **`schema/`**, **`forms/`**, **`csv/`**, **`presentation/`** (kiosk/presentation colors, seeded shuffle), **`utils/`**, **`hooks/`**. Prefer deep imports (e.g. **`/imports/lib/allocation/pledgeMatching`**) or the barrel for common symbols; not everything is re-exported from **`index.ts`**.
- **Non-UI hooks** (e.g. **`useInterval`**) live under **`imports/lib/hooks/`**.

## File storage
