# Allocation Project Documentation

## Project Overview
Architecture notes for this codebase: **why** non-obvious choices exist, not tutorials for things the code already shows.

## Publications and derived data

- **`memberThemesRefreshCoordinator`**: One debounced `MemberThemes` refresh per `themeId` is shared by `theme`, `organizations`, `members`, and **`member`** publications so concurrent subscribers do not multiply watchers or redundant full-table fetches. Kiosk `RemoteVoting` uses `useMember` + the `member` subscription instead of the full `members` list for the same reason.
- **`members` publication**: Membership is driven by a join through `MemberThemes`. When that join changes, the publication diffs IDs before touching the `Members` observer so dropped members emit **`removed`** immediately; the same transformer is reused so the client does not keep stale rows.
- **`organizations` publication**: A single org change can reorder or rescale the **global** chronological pledge walk for every org. Org `observe()` callbacks are wrapped in a **debounced full re-publish** so `matchedAmounts` and downstream org summaries stay coherent without re-implementing the walk per callback.
- **Refresh failures**: `catch` blocks in `themes.ts`, `organizations.ts`, and `members.ts` log through `publicationLog` (`LogCategories.Publication`, `mirrorToConsole: true`, `LogModels.*`) so broken refreshes show in server logs and in `Logs` when persist succeeds. Extra context that is not part of `LogContext` (e.g. a member id) goes under **`meta`**, not top-level fields.

## Pledge matching and leverage

- All matched-pledge math lives in **`imports/lib/pledgeMatching.ts`**. Do not re-implement `pledge.amount * matchRatio` elsewhere: it ignores the shared leverage pool, mixed `pledgeType` ratios, and partial-match truncation.
- **One pool per theme**: Eligible pledges across orgs share a single chronological queue (`createdAt`, then `_id`). Pool exhaustion can zero or partially match a later pledge regardless of org.
- **`calculatePledgeMatches` persists nothing**; it returns `matchedAmounts` + `remainingLeverage`. Publications recompute and pass **`matchedAmounts`** into transformers on each tick so edits/removals re-walk the queue without storing derived match state in Mongo.
- **Ratios**: `MatchPledge.pledgeType` selects `Theme.matchRatio` vs `Theme.inPersonMatchRatio`. Both are **total multipliers** (e.g. 2 ⇒ pledge + leverage bonus totals 2× pledge dollars from the pool perspective for matching logic as implemented).
- **Runner-up rule** is centralized in **`isOrgEligibleForLeverage`** for both queue construction and `pledgeTotalForOrg` — keep it in sync; do not duplicate.
- **`pledgedTotal` vs `pledgeMatchTotal`** in `ThemeTransformer`: raw pledged dollars vs total **leverage bonus consumed**; they differ whenever ratios ≠ 2, in-person pledges exist, or the pool truncated matches. **Why**: different surfaces need a footer that matches per-org rows vs a leverage equation that still closes.

## Feature flags vs runtime (presentation)

- **Flags** (Theme / PresentationSettings forms): “Is this capability allowed for this theme or presentation?” e.g. `Theme.inPersonPledgeActive`, `PresentationSettings.useKioskChitVoting`.
- **Runtime** (`PresentationPane`): “Is the live show doing this now?” e.g. `topupsActive`, `chitVotingActive`, `fundsVotingActive`, `currentPage`.
- **In-person pledges**: `/pledges/inperson/:themeId` is intended to work only when **both** `theme.inPersonPledgeActive` and `settings.topupsActive` are true; each false case gets its own messaging. **Server** `OrganizationMethods.pledge` enforces only the **theme flag** (same idea as standard pledges and runtime gating on the client) — keep that split deliberate.
- **In-person URL uses Mongo `_id`**: obscurity for a short window, not a secret; QR is **admin/settings only**, not on the presentation surface.

## Terminology (legacy cleanup)

- **pledge**, **crowdFavorite** (amount in `Organization.topOff`; `OrganizationMethods.crowdFavorite` runs once post–funds-vote, pre-pledges), **leverage** — see schema `// NOTE` on legacy names (`topOff`, `topupsActive`, etc.). **`CrowdFavoriteButton`** calls the method; do not hand-update `topOff`.

## Chit voting: two storage shapes

- **Manual** (`useKioskChitVoting` false): aggregate **`OrgChitVoteSummary`** on `Organization.chitVotes`; count beats weight; weight uses `Theme.chitWeight`.
- **Kiosk** (true): per-member **`ChitVote`** rows on `MemberTheme.chitVotes`; **`OrgTransformer`** aggregates. Do not conflate the two models.

## Logging (`imports/lib/logger.ts`)

- Methods under `imports/api/methods/` ship to the client; logger utilities must stay **outside `imports/server/`** so the bundle can load them.
- **Scope**: presentation / theme-scoped events only — **`themeId` is required** on `LogContext`. Cross-theme or non-presentation work uses **`console`** (or another path), not forced `themeId`.
- **Named loggers** in `loggers.ts` carry **`model`** as part of their contract; there is no generic “methods” logger. **`mirrorToConsole`** is per-call. **`Logs`** is server-insert only; clients use **`logs.purge`** / **`logs.recent`** with server-side validation.

## UI / tooling gotchas

- **`/imports/ui/components` barrel**: re-exports shared UI, forms, dialogs, cards, etc. It does **not** re-export **`Buttons/*`**, **`Toggles/*`**, or **`Inputs/*`** — import those from **`/imports/ui/components/Buttons/...`**, **`/imports/ui/components/Toggles`**, and **`/imports/ui/components/Inputs`**. The barrel exposes **`ConfirmationModal`** (`ConfirmDelete`) and **`CustomConfirmationModal`** (`CustomConfirm`, rich `content`) as distinct names.
- **TanStack Router pinned to `1.156.0`**: newer releases rely on `package.json` `exports` paths Meteor’s bundler does not resolve ([meteor#13520](https://github.com/meteor/meteor/pull/13520)); bumping the router without fixing that breaks the build.
- **Nested `$pull` on pledge arrays**: `_id` `autoValue` on embedded pledge schema must **not** `unset()` for non-`$push` operators or **`$pull` by `_id` becomes `$pull: { pledges: {} }`** and wipes the array.
- **MUI v9**: Checkbox/Switch use **`slotProps.input`**; Stack/Grid flex props belong in **`sx`**, not loose DOM props (MUI forwards only a small allowlist to the root).
- **React 19**: do not spread props that include **`key`**; pull `key` out and pass explicitly where MUI render props inject it.
- **`/pledges-overlay/$id`**: fullscreen stacks use **`position: fixed; inset: 0`** instead of **`100vw` / `100vh`** so width does not include the vertical scrollbar gutter (which forces horizontal scroll). **`PledgesOverlayDisplay`** sets **`overflow: hidden`** on **`document.documentElement`** and **`document.body`** while mounted. Pledge copy uses **`clamp(..., vmin, ...)`** and **`overflow-wrap: anywhere`** so scaled animation text stays inside the viewport.

## Meteor / tests / scripts

- **ValidatedMethod**: calling **`.callAsync()` from inside another server method** can hang — prefer direct collection access in that case.
- **Root `scripts/`**: files with top-level side effects can run on Meteor startup; keep heavy scripts under `imports/` or gate on env.
- **Tests**: **`resetDatabase`** uses per-collection **`removeAsync({})`** instead of **`dropDatabase()`** to avoid Accounts races and slow index rebuilds. **`npm test`** runs **server** Mocha by default (no headless browser) unless you set **`TEST_BROWSER_DRIVER`** for client **`*.test.*`** files (see [meteor-mocha README](https://github.com/meteortesting/meteor-mocha/blob/master/README.md#run-app-tests)). Suites that need **full Mongo / methods / Collection2** (e.g. **`resetDatabase`**, **`ValidatedMethod.callAsync`**, schema **`autoValue`**) live under **`server/api/**`** (mirrors **`imports/api/methods`** and **`imports/api/db`**) so they never load in the client harness. Isomorphic **`imports/**/*.test.ts`** remain for logic that is safe on both sides (e.g. **`imports/lib/**`**). Tests that import **`imports/server/**`** (e.g. mocks that pull **`OrgTransformer`**) must live under **`server/`** or **`imports/server/`**, not **`imports/lib/`**, or the **client** test bundle will fail to load.
- **Publication load harness**: env-tunable scale in **`imports/server/test-support/publicationPipelineSimulation.ts`**; tests live in **`server/publicationPipelineSimulation.test.ts`** (server-only directory so the harness is not pulled into the **client** test bundle).

## Types and collections

- **`meteor/aldeed:collection2`** / **`meteor/aldeed:collection2/static`**: Meteor package ids, not npm; TypeScript does not ship typings for them. **`imports/types/collection2.d.ts`** declares those modules (side-effect imports) and augments **`Mongo.Collection`** with **`attachSchema`**. Same pattern as other **`imports/types/*.d.ts`** shims for Meteor or non-typed packages.
- **Generated types**: `npm run generate-types` → `imports/types/schema.ts`; collection vs embedded schemas differ (e.g. **`_id`** on collection-attached schemas only). **Embedded** `OrgChitVoteSummary` ≠ **`ChitVote`** on member themes — different purposes; name collisions across files are a footgun.
- **`Members` minimongo type is `MemberWithTheme`** (`MemberData` + optional **`theme`** from `members` / `member` publications) so client reads match published shape. **`MemberData`** remains the generated **Member** schema type from SimpleSchema. Import **`MemberWithTheme` from `/imports/api/db`** in UI and hooks — not from `imports/server/transformers/memberTransformer` (that path is server-architecture-only in Meteor and breaks the client test bundle).

## Kiosk / member login

- **`KioskVotingProvider` / `useKioskVoting`**: one session object for **both** funds allocations and chit tallies + saves + `member` + `unsetUser` (not funds-only). **`MemberLoginRequired`** vs **`RemoteVoting`**: URL without `member` uses the shared login form for chit/funds; pledges with a `member` query use **`RemoteVoting`** only.
- **`useFindMemberByCode`**: exposes **`findMemberByCode`**, **`findMemberById`**, and **`members` / `membersLoading`** from one **`members`** subscription so callers do not stack **`useMembers`** and duplicate the sub. **`MemberCodeLoginForm`**: folder + barrel under **`/imports/ui/components`**; shared kiosk + in-person login (**`notFoundStyle`** differs).
- **Non-UI hooks** (e.g. **`useInterval`**) live under **`imports/lib/hooks/`**.

## File storage

- **`ostrio:files`**, uploads under **`public/.uploads`**, URLs use **`HOST_URL`** where absolute links matter (e.g. email).
